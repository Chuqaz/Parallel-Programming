## Анализ и выводы
С ростом количества процессов эффективность снижается.Это особенно видно при сравнении результатов для 8-ми и 6-ти процессов соответственно.Также стоит отметить ненужность использования большого количества процессов при малых вычислениях из-за больших накладных расходов, что мы можем наблюдать для размера матрицы 200 для 8-ми и 6-ти процессов (для шести процессов операция выполнилась быстрее).Однако для больших объёмов данных стоит использоват наибольшее число процессов.Программа успешно распараллелена.
